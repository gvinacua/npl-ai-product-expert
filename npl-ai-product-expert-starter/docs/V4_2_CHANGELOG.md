# V4.2 Changelog — Adoption/Product Balance

This version corrects the expert's stance on AI adoption.

Changes:
- Replaced the binary “licenses are not transformation” framing with a more mature adoption-to-product thesis.
- Added explicit distinction between unmanaged tool dumping and adoption-with-operating-model.
- Added knowledge pack `17_ai_operating_model_and_adoption_balance.md`.
- Added BBVA-style operating model nuance: broad access, safe platforms, internal GPT marketplaces, AI factory / hub-and-spoke support, proficiency curves, communities and reusable skills.
- Added Brainfood AI Operating Model concepts as a source to be used carefully and verified before client delivery.
- Updated expert style: concise, but less dogmatic; sharper because it can hold two truths at once.

New expert thesis:

AI transformation does not come from licenses alone, nor from isolated AI products alone. It comes when bottom-up adoption, product discipline, operating model, governance and workflow redesign reinforce each other.
