# v5 Changelog — Jobs-to-be-done UX reset

## Purpose
v5 resets the product experience around three core jobs-to-be-done instead of exposing many modes and controls on one screen.

## New primary workflows
1. **Build a learning asset** — create sessions, programmes, source packs and client-facing training material.
2. **Talk with the expert** — interact with the expert in text or voice for debate, panel thinking, POV building and sparring.
3. **Deliver / activate** — prepare content for use with an audience via spoken blocks, live Q&A support, voice delivery and operator-ready scripts.

## Transversal layers
Quality, evidence and cost are not standalone primary jobs. They appear as lightweight signals and checks inside each workflow.

## UX changes
- Replaced mode-first interface with three job cards.
- Added focused action choices inside each job.
- Hid advanced controls behind an expandable panel.
- Kept cost proxy, deep model and web search controls, but made them secondary.
- Kept interactive voice expert in the “Talk with the expert” workflow.
- Kept ElevenLabs block audio in the “Deliver / activate” workflow.
- Added a compact quality check panel that audits the current output without forcing the user into a separate audit mode.

## Product direction
This version is not a visual polish pass. It is a structural UX reset. Future UI work should further simplify around outcomes, saved assets, source cards and delivery formats.
