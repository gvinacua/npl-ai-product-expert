# v4.4 Changelog — Evidence Discipline Upgrade

## Purpose
Strengthen the credibility of the NPL AI Product Expert by making evidence, links, named-case verification and claim status more explicit across modes.

## What changed

### Global evidence discipline
- Named-company examples must not be called confirmed unless the specific claim is supported by local knowledge or web search.
- Each real company case should include source/link if available, what is confirmed, what is inferred, what remains to verify, and how it supports the lesson.
- Vendor case studies are useful but should be labeled as vendor-provided where relevant.
- Avoid vague language such as “leading banks show” unless a source is named or the statement is marked as NPL hypothesis.

### Source-Grounded Lesson Builder
- Requires evidence anchors before the lesson.
- Classifies claims as confirmed, emerging, hypothetical, vendor-provided, inferred or to verify.
- Prefers fewer strong examples over many weak ones.
- Adds explicit “claims to verify before client delivery.”

### Point-of-View Builder
- Requires 3–5 evidence anchors when possible.
- Each anchor should include source/link if available, what it supports, what it does not prove, and confidence level.
- Adds senior-audience challenge question.

### Other modes
- Speaker, Training, Panel, Frontier, Research and Audit modes now reinforce evidence notes and source links where relevant.

## Quality target
The expert should not only sound sharp. It should make clear what is known, what is inferred, what is emerging, and what still needs verification before client use.
