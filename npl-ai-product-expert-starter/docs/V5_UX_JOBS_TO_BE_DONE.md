# v5 UX structure — Jobs to be done

## Core decision
The app should not feel like a technical console with many exposed modes. It should feel like a focused internal expert tool organized around user jobs.

## Job 1: Build a learning asset
User intent: “I need to prepare content for a programme, training objective, proposal or client session.”

Typical outputs:
- session/module
- programme/proposal section
- source pack
- slide outline
- exercise
- Q&A bank
- claims to verify

Embedded checks:
- quality score
- evidence status
- cost estimate
- client-readiness status

## Job 2: Talk with the expert
User intent: “I want to discuss, challenge, explore, debate or ask questions with the AI expert.”

Formats:
- text answer
- point-of-view builder
- panel answer
- sparring
- live voice conversation

Embedded checks:
- confidence and verification notes
- answer length discipline
- NPL provisional position when contested

## Job 3: Deliver / activate
User intent: “I want to use this with an audience.”

Formats:
- spoken blocks
- ElevenLabs block audio
- browser voice
- live Q&A support
- operator-ready delivery pack
- presenter notes / slide cues

Embedded checks:
- voice cost estimate
- delivery readiness
- evidence warnings
- safe-to-say / verify-first notes

## What is not a primary job
- audit
- cost dashboard
- mode selection
- voice as a standalone product
- research scout as a standalone product

These are supporting capabilities inside the three jobs.
