# Evaluation Playbook v1

Use this whenever you want to know whether the NPL AI Product Expert is actually good.

## Workflow
1. Generate a talk, lesson or briefing.
2. Click `Audit current output`.
3. Use the deep model if available.
4. Review scores and weak points.
5. Ask for a rewrite using the audit recommendations.
6. Run the audit again.
7. Only use client-facing material once it reaches `Strong with edits` or `Client-ready`.

## Minimum bar
Client-facing material should not be accepted if it:
- sounds like a generic AI transformation talk;
- lacks financial-services specificity;
- lacks product lifecycle detail;
- has no governance/risk implications;
- contains named-company or regulatory claims without verification notes;
- has no exercise or audience interaction;
- does not contain at least one fresh or non-obvious idea.

## Best quality prompt
```
Audit this output as if NPL's reputation depends on it. Would a senior banking or insurance audience learn anything non-obvious? Score it strictly. Then rewrite the weakest section.
```

## Best freshness prompt
```
Upgrade this talk with 10 frontier signals from AI agents, voice interfaces, AI product evaluation, agent security, financial-services AI and regulation. Separate verified facts from weak signals and speculation.
```
